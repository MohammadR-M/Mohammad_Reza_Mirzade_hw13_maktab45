• سازگاری سطح بالا با HTTP / 1.1 - روش ها ، کدهای وضعیت ، URI ها و هدرها.
  • بهبود سرعت بارگذاری صفحه از طریق:
    • فشرده سازی request header ها. از مشخصات HPACK برای فشرده سازی هدرها استفاده میکند.
    • استفاده از پروتکل باینری.
    • HTTP/2 Server Push: اطلاعات اضافی را برای مشتری ارسال میکند که مشتری درخواست نکرده است ولی در آینده پیش بینی میشود.
    • برقراری چندین کانکشن با سرور.
